Architecture profile

Architecture Formalism

This profile mandates NAFv4 as the conceptual foundation for architecting in NATO.

Architecture Management Application

Mandatory

Organisations will have to select one of the two agreed Meta Models either either Archimate from Open Group or UAF from OMG.

  • C3B ac322-n2018-0002 - NATO Architecture Framework
  • Open Group c197 - ArchiMate 3.1 Specification
  • OMG formal/2017-12-02 DMM - Unified Architecture Framework 1.0 (UAF) Domain Meta Model (DMM)

Optional

  • ISO/IEC/IEEE 42010 - Systems and software engineering -- Architecture description

Architecture Standard Language

The following standards describe both a modelling language and a notation for developing models. UAF and Archimate for modelling NAF 4 compliant architecture descriptions.

Architecture Management Application

Mandatory

Depending on the meta Model uses the following specifications apply

  • OMG formal/2017-12-02 - Unified Architecture Framework 1.0 (UAF) Domain Meta Model (DMM)
  • Open Group c197 - ArchiMate 3.1 Specification

Optional

  • Eclipse Capella 1.4 - Arcadia/Capella
  • IDEF IDEF0 - Function Modeling Method
  • IDEF IDEF1X - Data Modelling Method
  • OMG formal-2012-05-10 - Service Oriented Architecture Modeling Language (SOAML)

Architecture Process

When developing architectures the adapted ADM methodology from TOGAF as described in NAF 4 chapter 2 should be followed. It is also recommended to be familiar with ISO 42020 as it covers aspect not included in NAF 4.

Architecture Management Application

Mandatory

  • C3B ac322-n2018-0002 - NATO Architecture Framework

Recommended

  • ISO/IEC/IEEE 42020 - Enterprise, systems and software - Architecture processes

Chapter 2 in NAF should always be followed42020 may be used as well

Architecture Evaluation

ISO 42030 describes how architecture evaluations are performed for many reasons, including – determining if a system of interest has been or is being architected in such a way that it fulfils its intended purpose, evaluating the architecture’s effectiveness and suitability towards particular needs of stakeholders, identifying risks for mitigation and identifying opportunities for its improvement. Stakeholders involved in architecture governance are often accountable for these and they have a need to conduct or commission recurring architecture evaluations which can be executed in a systematic fashion.

Architecture Management Application

Mandatory

For evaluation of Architecture Products use ISO/IEC/IEEE 42030 an ATAM as evaluation method is recommended

  • ISO/IEC/IEEE 42030 - Enterprise, systems and software - Architecture Evaluation

Recommended

Use ATAM as evaluation method for architecture products.

  • SEI cmu-sei-2000-tr-004 - ATAM: Method for Architecture Evaluation

Mandatory

If assessment of the applied architecting process is required

  • ISO/IEC/IEEE 42020 - Enterprise, systems and software - Architecture processes

Architecture Product Exchange

Architecture products shall be exchanged in accordance with the ArchiMate Model Exchange File Format using a pivot meta model

Architecture Management Application

Mandatory

  • Open Group c19c - ArchiMate Model Exchange File Format for the ArchiMate Modeling Language 3.1

This requires a new guidance document to be developed by the ACaT.

Architecture Management

Ensure execution of directives for development of the architectures, to ensure that the development runs according to these directives, to the expected timetables, to the assigned budgets, and that the architecture satisfies its objectives.

Architecture Management Application

Mandatory

  • ISO/IEC/IEEE 42020 - Enterprise, systems and software - Architecture processes

For service quality aspects - partially or totally compliant.

Architecture Governance

Establish standards and policies related to one or more architectures of interest and their development, and to monitor and facilitate the alignment of the architecture(s) to stakeholder concerns, policies and standards, including organizational and environmental constraints.

Architecture Management Application

Mandatory

  • ISO/IEC/IEEE 42020 - Enterprise, systems and software - Architecture processes

For methodology and quality aspects - partially or totally compliant.
