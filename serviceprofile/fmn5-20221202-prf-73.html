Title
Traffic Flow Confidentiality Protection Profile
Description
The Traffic Flow Confidentiality Protection Profile provides standards and guidance for implementing IPSEC based protection for data traffic.

Standards

Obligation: MANDATORY, Lifecycle: CANDIDATE

These are standards to implement protection profiles needed for IPSec.

  • IP Encapsulating Security Payload (ESP) (RFC 4303:2005)
  • IKE and IKEv2 Authentication Using the Elliptic Curve Digital Signature Algorithm (ECDSA) (RFC 4754:2007)
  • Elliptic Curve Groups modulo a Prime (ECP Groups) for IKE and IKEv2 (RFC 5903:2010)
  • Internet Key Exchange Protocol Version 2 (IKEv2) (RFC 7296:2014)
  • The Use of Galois/Counter Mode (GCM) in IPsec Encapsulating Security Payload (ESP) (RFC 4106:2005)
  • Using HMAC-SHA-256, HMAC-SHA-384, and HMAC-SHA-512 with IPsec (RFC 4868:2007)
  • Suite B Cryptographic Suites for IPsec (RFC 6379:2011)
  • Algorithm Implementation Requirements and Usage Guidance for the Internet Key Exchange Protocol Version 2 (IKEv2) (RFC 8247:2017)
